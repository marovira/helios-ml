# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.0.x   | Yes       |
| < 2.0   | No        |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, send a description of the issue to
[maroviragalvez@outlook.com](mailto:maroviragalvez@outlook.com). Include as much detail
as possible: the affected component, steps to reproduce, and any potential impact.

You can expect an acknowledgement within 48 hours. Once the issue has been assessed, you
will be kept informed of the progress toward a fix and the timeline for a release. We ask
that you give us a reasonable amount of time to address the issue before any public
disclosure.
