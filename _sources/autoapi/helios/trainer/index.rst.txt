.. py:function:: find_last_checkpoint(root: pathlib.Path | None) -> pathlib.Path | None

   Find the last saved checkpoint (if available).

   The function assumes that checkpoint names contain ``epoch_<epoch>`` and
   ``iter_<iter>`` in the name, in which case it will return the path to the checkpoint
   with the highest epoch and/or iteration count.

   :param root: the path where the checkpoints are stored.

   :returns: The path to the last checkpoint (if any).


.. py:class:: Trainer(run_name: str = '', train_unit: TrainingUnit | str = TrainingUnit.EPOCH, total_steps: int | float = 0, valid_frequency: int | None = None, chkpt_frequency: int | None = None, print_frequency: int | None = None, accumulation_steps: int = 1, enable_cudnn_benchmark: bool = False, enable_deterministic: bool = False, early_stop_cycles: int | None = None, use_cpu: bool | None = None, gpus: list[int] | None = None, random_seed: int | None = None, enable_tensorboard: bool = False, enable_file_logging: bool = False, capture_warnings: bool = False, enable_progress_bar: bool = False, chkpt_root: pathlib.Path | None = None, log_root: pathlib.Path | None = None, wandb_args: helios.core.loggers.WandbArgs | None = None, src_root: pathlib.Path | None = None, import_prefix: str = '', print_banner: bool = True)

   Automates the training, validation, and testing code.

   The trainer handles all of the required steps to setup the correct environment
   (including handling distributed training), the training/validation/testing loops, and
   any clean up afterwards.

   :param run_name: (optional) name of the current run.
   :param train_unit: (optional) the unit used for training.
   :param total_steps: (optional) the total number of steps to train for.
   :param valid_frequency: (optional) frequency with which to perform validation.
   :param chkpt_frequency: (optional) frequency with which to save checkpoints.
   :param print_frequency: (optional) frequency with which to log.
   :param accumulation_steps: (optional) number of steps for gradient accumulation.
   :param enable_cudnn_benchmark: (optional) enable/disable CuDNN benchmark.
   :param enable_deterministic: (optional) enable/disable PyTorch deterministic.
   :param early_stop_cycles: (optional) number of cycles after which training will stop if
                             no improvement is seen during validation.
   :param use_cpu: (optional) if true, CPU will be used.
   :param gpus: (optional) IDs of GPUs to use.
   :param random_seed: (optional) the seed to use for RNGs. Defaults to the value
                       returned by :py:func:`~helios.core.rng.get_default_seed`.
   :param enable_tensorboard: (optional) enable/disable Tensorboard logging.
   :param enable_file_logging: (optional) enable/disable file logging.
   :param capture_warnings: (optional) enable/disable warnings capture in the log.
   :param enable_progress_bar: (optional) enable/disable the progress bar(s).
   :param chkpt_root: (optional) root folder in which checkpoints will be placed.
   :param log_root: (optional) root folder under which each active logger creates
                    its own subfolder.  Required when ``enable_file_logging``,
                    ``enable_tensorboard``, or ``wandb_args`` is set.
   :param wandb_args: (optional) W&B configuration.  Setting this enables W&B
                      logging and requires ``log_root`` to be set.  See
                      :py:class:`~helios.core.loggers.WandbArgs` for the available keys.
   :param src_root: (optional) root folder where the code is located. This is used to
                    automatically populate the registries using
                    :py:func:`~helios.core.utils.update_all_registries`.
   :param import_prefix: (optional) prefix to use when importing modules. See
                         :py:func:`~helios.core.utils.update_all_registries` for details.
   :param print_banner: (optional) if true, the Helios banner with system info will be
                        printed.

   .. py:method:: register_plugin(plugin: helios.plugins.Plugin) -> None

      Register a plug-in with the trainer.

      The plug-in is registered using its ``plug_id``.

      .. note::
          :py:meth:`~helios.plugins.plugin.Plugin.configure_trainer` and
          :py:meth:`~helios.plugins.plugin.Plugin.configure_model` are called
          automatically by the trainer before training/testing.

      :param plugin: the plug-in to register.

      :raises KeyError: if a plug-in with the same ID is already registered.
      :raises ValueError: if the plug-in conflicts with an already-registered plug-in's
          :py:class:`~helios.plugins.plugin.UniquePluginOverrides`.



   .. py:property:: queue
      :type: torch.multiprocessing.Queue | None


      Return the multi-processing queue instance.

      .. note::
          If training isn't distributed or if `torchrun`, then `None` is returned
          instead.
